Getting Started

System Requirements

  • Node.js 16+ and Python 3.9+
  • PostgreSQL database with intelligence tables
  • Docker for containerized deployment
  • Required environment variables configured
  • API access to Dutch data sources (where available)

Initial Setup

Configure Environment Variables

Set up your .env file with Dutch data source configurations:

# Dutch Data Sources CBS_API_KEY=your_cbs_api_key DNB_API_KEY=your_dnb_api_key KADASTER_API_KEY=your_kadaster_api_key # Intelligence System MARKET_INTELLIGENCE_ENABLED=true INTELLIGENCE_EXECUTOR_WORKERS=4 FORECAST_HORIZON_DAYS=90

Initialize Database Tables

Ensure all intelligence tables are created using the provided schema.

Note: The system requires 8 additional database tables for market intelligence data storage.

Start Services

Launch the market intelligence services:

docker-compose up -d # Or start services individually npm start python backend/agents/utils/dutch_market_intelligence_executor.py

Access the Interface

Navigate to the Dutch Market Intelligence section in your MortgageAI dashboard.

Trend Analysis Workflow

Data Preparation

Ensure sufficient data points (minimum 10) for reliable trend analysis.

Statistical Analysis

The system performs multiple statistical tests:

  • Linear regression analysis
  • Statistical significance testing (p < 0.05)
  • Confidence interval calculation (95%)
  • R-squared goodness of fit

Trend Classification

Trends are automatically classified based on statistical properties:

  • Upward: Positive slope with high confidence
  • Downward: Negative slope with high confidence
  • Stable: Low volatility, minimal slope
  • Volatile: High variability, low predictability

Key Factors Identification

System identifies factors contributing to observed trends.

Results Interpretation

Review trend strength, confidence levels, and statistical significance.

Best Practice: Use at least 30 data points for seasonal analysis and 90+ points for robust trend identification.

Troubleshooting Guide

Common issues and solutions for the Dutch Market Intelligence system.

Data Collection Issues

Issue: "Failed to collect market data"
Possible Causes:
  • Missing or invalid API credentials
  • Rate limit exceeded
  • Data source temporarily unavailable
  • Network connectivity issues
Solutions:
  • Verify API keys in environment variables
  • Check rate limiting settings
  • Retry after waiting period
  • Contact data provider support

Analysis Failures

Issue: "Insufficient data for analysis"
Possible Causes:
  • Not enough historical data points
  • Poor data quality scores
  • Missing required metrics
Solutions:
  • Extend date range for data collection
  • Select different data sources
  • Lower quality thresholds temporarily

Performance Issues

Issue: Slow task execution
Optimization Steps:
  • Increase worker pool size
  • Enable caching mechanisms
  • Reduce data collection scope
  • Schedule tasks during off-peak hours
